Section 1: Portfolio Carousel Redesign

Replacing the clunky carousel with a better interaction pattern. Which approach do you prefer?

A

Improved Carousel (Keep Pattern, Fix UX)

Keep the horizontal carousel but fix everything wrong with it:

  • Smooth snap scrolling with CSS scroll-snap-type
  • Arrow buttons (left/right) for desktop navigation
  • Keyboard navigation (arrow keys)
  • Proper dot indicators synced to scroll position
  • Touch swipe on mobile with momentum
  • Modal with focus trap and Escape to close
B

Filterable Grid (Recommended)

Replace the carousel with a responsive grid + category filter tabs:

  • Filter tabs: All / Web / Mobile / Design
  • Responsive grid (3 cols desktop → 2 tablet → 1 mobile)
  • Animated card entrance with Framer Motion layoutId
  • Hover reveals project summary overlay
  • Click opens detail modal (same as current, but with focus trap + keyboard)
  • No scroll sync bugs — everything visible at once

Modal improvements (applies to both)

Current Issues

  • No focus trap — Tab key escapes modal
  • No Escape key to close
  • Background still scrollable
  • No transition animation

Proposed Fix

  • Focus trap on open, restore on close
  • Escape key closes modal
  • overflow: hidden on body when open
  • Scale + fade entrance animation
  • Click outside (scrim) to dismiss
