How should we approach this?

Three strategies — same scope, different trade-offs. Pick the one that fits your goals.

A

Surgical Fixes + New Sections (Recommended)

Keep the current design DNA. Fix what's broken (carousel, skills categorization, accessibility, performance), add polish (Inter font, refined spacing), and bolt on the 4 new sections in the same neon/glassmorphism style.

Pros
  • Lowest risk — preserves your existing work
  • Fastest to complete
  • No visual regression for returning visitors
Cons
  • Won't fix structural CSS issues (global styles)
  • Some components stay complex (ProfileCard)
B

Design Refresh

Same layout structure, elevated execution. Rework the styling system (CSS variables → Tailwind tokens), replace the carousel with a proper grid/masonry layout, upgrade all animations to be more subtle and performant, add new sections.

Pros
  • Cleaner codebase going forward
  • Better maintainability
  • Portfolio section gets a proper redesign
Cons
  • More changes = more risk of regressions
  • Takes longer to implement
C

Full Rebuild

Start fresh with a new design system. Redesign from scratch using a cohesive Tailwind-first approach, proper component architecture, full accessibility baked in from day one, and all new sections included.

Pros
  • Cleanest result — no legacy baggage
  • Best long-term maintainability
  • Accessibility built-in, not bolted on
Cons
  • Highest effort — essentially a new project
  • Risk of losing the "character" of current design
  • Takes significantly longer
