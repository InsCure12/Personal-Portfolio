Section 4: Performance & Accessibility

Making the site fast and usable for everyone.

Performance Optimizations

WebGL Particles

Currently 200 particles on all devices. Proposal:

  • Desktop: keep 200 particles
  • Mobile: reduce to 80 particles (saves battery, smoother scrolling)
  • Add prefers-reduced-motion check — disable particles entirely if set

Bundle Size

Three.js + @react-three/fiber are bundled but never used in the app. Removing them saves ~200KB+ gzipped.

Image Optimization

  • Add loading="lazy" to project screenshots
  • Add explicit width / height to prevent CLS (layout shift)
  • Use aspect-ratio CSS for responsive image containers

Font Loading

  • Preload Inter variable font
  • Use font-display: swap to avoid invisible text
  • System font fallback sized to match Inter metrics

Accessibility Improvements

Navigation

  • Skip-to-main-content link
  • Keyboard nav for header menu
  • Visible focus rings on all interactive elements
  • Focus trap in mobile drawer

Portfolio Modal

  • Focus trap when open
  • Escape key closes
  • Restore focus on close
  • aria-modal="true"
  • Body scroll lock

Images & Icons

  • Alt text on all meaningful images
  • Decorative images: alt=""
  • Replace emoji with SVG + aria-label
  • Icon buttons get aria-label

Semantic HTML

  • Proper heading hierarchy (h1→h6)
  • Use <main>, <nav>, <section>
  • ARIA landmarks for screen readers
  • Reduced motion support on all animations
